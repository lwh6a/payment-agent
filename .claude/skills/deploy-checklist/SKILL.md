# 发版检查单生成

**核心原则**：只输出本次改动实际涉及的检查项。没改的中间件不出现，不生搬固定清单。

## 生成逻辑

```
输入：代码 diff（或改动范围描述）
Step 1 扫描改了什么：新增/修改了哪些接口、表、配置、MQ、Job、依赖、外部调用
Step 2 按映射表逐条生成检查项：每条检查项必须带本次改动的具体内容
       （不是"检查 SQL"，而是"检查 xxx.sql 是否已执行"）
Step 3 输出 SIT 检查单（环境就绪确认）+ PROD 检查单（SIT 全部项 + 生产特有风险项）
```

## 改动 → 检查项映射表（AI 内部对照）

### 代码层面
| 改动类型 | SIT 检查项 | PROD 额外检查项 |
|---|---|---|
| 新增/修改接口 | 接口文档已更新 | 性能预期，是否需要限流 |
| 接口权限变更 | 鉴权注解正确、权限点已注册 | 生产权限与 SIT 一致 |
| 新增/修改错误码 | 已与前端/QA 同步 | 提示文案配置覆盖就绪 |
| 新增 RPC/HTTP 下游调用 | 下游 SIT 可用、超时配置合理 | 生产下游已发布对应版本 |

### 数据层面
| 改动类型 | SIT 检查项 | PROD 额外检查项 |
|---|---|---|
| 新增表/加字段 | DDL 已执行、历史数据默认值正确 | DBA 审核、生产 DDL 已执行、大表在线 DDL、回滚方案 |
| 修改索引 | SIT 索引已创建 | 大表创建索引避开高峰期 |
| 数据迁移脚本 | SIT 已执行并验证 | 迁移窗口、数据备份、回滚方案 |
| 枚举值变更 | SIT 已更新 | 存量数据是否兼容旧枚举值 |

### 配置层面
| 改动类型 | SIT 检查项 | PROD 额外检查项 |
|---|---|---|
| Nacos 新增/修改配置 | SIT namespace 已配置（附具体 key、旧值→新值） | 生产 namespace 已同步、与 SIT 差异已确认 |
| application.yml 变更 | 各环境 profile 已对应修改 | — |

### 中间件层面
| 改动类型 | SIT 检查项 | PROD 额外检查项 |
|---|---|---|
| 新增 MQ Topic/Consumer | Topic 已创建（附名称）、消费逻辑幂等 | 生产 Topic 已建、消费组隔离、死信队列 |
| 新增 XXL-JOB 任务 | 任务已配置（附 handler + cron）、SIT 已调度验证 | 生产已配置、cron 与 SIT 一致、告警已配置 |
| 新增 Redis key | 命名规范、无冲突 | 生产 key 不冲突 |

### 依赖与环境
| 改动类型 | SIT 检查项 | PROD 额外检查项 |
|---|---|---|
| 新增 Maven 依赖 | 版本与工程基线一致、无冲突 | — |
| 新增外部 HTTP 调用 | SIT 目标可达、超时/重试已配置 | 网络白名单已开通（附域名/IP） |

### 发布策略（PROD 特有）
灰度方案 / 回滚方案（代码+数据+配置三层）/ 监控告警（新接口与 Job）/ 上线窗口（时间、操作人、值班人）/ 上下游协同（依赖服务上线顺序）/ 回归验证（场景、验证人、标准）。

## 输出格式

按改动分组输出 Markdown 勾选清单，示例：

```markdown
## 📋 SIT 发版检查单 — <需求/Bug 编号>
### 数据库
- [ ] DDL: `pay_order_extend.sql` — 新增 `xxx` 字段，历史数据默认值已确认
### MQ
- [ ] 新增 Topic: `pay_callback_topic` — 已申请创建
### 代码
- [ ] 已合并到部署分支；自测：主流程 ✅ / 异常 ✅ / 回归 ✅
```

PROD 检查单 = SIT 全部项 + 发布策略项，数据库变更必须包含 DBA 审核与回滚方案。

## 特殊情况

- **纯代码逻辑修复**：SIT = 合并 + 自测 + 提测说明；PROD = 合并 + 回归验证 + 上线窗口。
- **仅配置变更不发版**：SIT = 配置已更新并验证生效；PROD = 与 SIT 差异确认 + 动态刷新即可。
- **回滚发版**：清单反向生成——提示回滚哪些 DDL、配置、代码。

## 自检（不对外输出）

- 检查单是否只含本次改动涉及的项？
- 每条是否带具体内容？
- PROD 是否包含 SIT 全部项 + 生产特有项？
- 数据库变更是否有 DBA 审核 + 回滚方案？外部调用是否有白名单项？
